# 硬规则与反模式 — deep-research

所有规则按"在哪拦截"标注，违规即降级。

---

## 阶段1 规则

| 规则 | 说明 | 在哪拦截 |
|:----|:-----|:---------|
| **章节约束** | quick 6-8 章 / standard 8-10 章 / deep 10-12 章 | 阶段1 oracle |
| **可搜索性** | 每个子问题必须能通过 Exa + 免费源（DuckDuckGo/Semantic Scholar/GDELT/Wikipedia）独立搜索 | 阶段1 oracle |
| **免费源补强** | Exa 结果不足时触发 Step 1.5，补强 URL 必须入 Scrapling | 阶段2a Step 1.5 |
| **对比视角** | 至少 1 个反方观点子问题 | 阶段1 oracle |
| **子节结构完整性** | Oracle 必须为每章定义 sections[]，deep 3-6 节，standard 2-4 节，quick 1-2 节 | 阶段1 oracle |

## 阶段2 规则

| 规则 | 说明 | 在哪拦截 |
|:----|:-----|:---------|
| **Scrapling 不可跳过、不可替代** | Exa 后必须批量抓取全文。禁止从 Exa 摘要片段提取数据。三种工具全失败则标记"来源稀缺"跳过 | ⚡ 阻断点（Step 2→Step 3） |
| **结构化事实** | 数据池使用 JSON facts[] 数组格式，非散装文本 | 阶段2a 输出 |
| **跨源矛盾标记** | 不同来源矛盾数据在 controversies[] 数组中对齐 | 阶段2a 数据池 |
| **来源稀缺** | 连续 3 不同域名 404 → 2 分钟上限 | 阶段2a |
| **不重复** | 每个新子问题前检查数据池已有内容 | 阶段2a Step 3 |

## 阶段3 规则

| 规则 | 说明 | 在哪拦截 |
|:----|:-----|:---------|
| **标题自解释** | 标题不含模拟编号（C1/ch-01 等）。grep 检查 `^##.*ch-\|^##.*C1` | 阶段3c 验收 |
| **子节编号体系分离** | 子节标题（二级 / `###`）不得使用汉字数字编号体系（一、二、三），必须使用阿拉伯数字 x.x 格式（1.1, 1.2…） | 章节 agent prompt + 阶段3c 验收 grep `^### [一二三四五六七八九十]` |
| **时间戳硬编码** | 文件名和报告尾时间必须 `date` 获取 | 阶段3c 验收 |
| **章节 agent 不写最终文件** | 写临时文件，装配 agent 拼装 | 阶段3b Step 2 |
| **跨源一致** | 同一数据在不同章节数值一致 | 阶段3c 装配检查 |
| **清理** | 调研结束后删除 TMPDIR | 阶段3b Step 5 |

## 反模式表

| 反模式 | 在哪拦截 |
|:-------|:--------|
| 数值量级错误（亿/billion 差 10x） | 阶段3b 章节 agent 自检 + 阶段3a 预检 |
| 完美平滑趋势（虚假插值） | 阶段3b 章节 agent 自检 |
| 来源混淆（出货量 vs 零售量） | 阶段3b 章节 agent 自检 |
| 数据不足硬撑行数 | 阶段3a 预检阻止 + 阶段3b 段落数约束 |
| **Scrapling 被 Exa 片段替代** | **阶段2a ⚡ 阻断点** |
| **内部编号泄露到正文**（如出现 "以下是C1的分析"） | **阶段3b prompt + 阶段3c grep 检查** |
| **子节使用一、二、三编号**（应与章编号体系分离） | **阶段3b prompt + 阶段3c 验收 grep `^### [一二三四五六七八九十]、`** |
| **时间戳硬编码** | **阶段3c 验收** |
| **章节 agent 写最终文件**（并行竞态） | **阶段3b 临时文件方案** |
| **并行章节数据不一致** | **阶段3c 装配一致性检查** |
| 大纲超约束 | 阶段1 硬约束章节数 |
